# REGISTRE D'ACTIVITÉS PHYSIQUES

## Objet
Index structuré du référentiel d'activités physiques du projet.  
Facilite la sélection et l'analyse lors de la planification hebdomadaire.

**Mainteneurs** : PS-PLAN-08 (sélection), PS-SCR-06 (persistance).  
**Révision** : à chaque création ou modification d'une activité.

## 2. Vue par intensité (RPE) et charge

### Très léger (RPE 1–2)
- RUN-30-RECUP-V1 (récupération),
- FB-30-MOBI-RENFO-V1 (mobilité).

**Usage** : J+1 après effort, digestion, fatigue, réduction de charge.

### Léger-modéré (RPE 3–4)
- RUN-40-EF-V1 (endurance fondamentale),
- RUN-60-SL-V1 (sortie longue, base),
- FB-30-MOBI-RENFO-V1 (mobilité).

**Usage** : Semaines normales, accumulation de volume, séance tampon.

### Modéré (RPE 5–6)
- FB-30-BASE-V1 (base renfo),
- RUN-40-FARTLEK-V1 (base EF + accélérations),
- RUN-40-TEMPO-FRACTIONNE-V1 (tempo),
- FB-30-DENSITÉ-V1 (densité).

**Usage** : Séances qualité maîtrisées, phase de stabilisation.

### Soutenu à intense (RPE 6–7)
- RUN-45-SEUIL-V1 (seuil),
- RUN-40-TEMPO-FRACTIONNE-V1 (blocs tempo),
- FB-40-FORCE-V1 (force),
- FB-30-DENSITÉ-V1 (densité).

**Usage** : Phases consolidation/progression. À limiter en diète active. Jamais deux le même jour.

## 3. Vue par contexte de phase

### Phase 1 (Établissement)
- RUN-30-RECUP-V1 ✅

### Phase 3 (Progression)
- RUN-30-RECUP-V1 ✅
- RUN-40-EF-V1 ✅ (base)
- RUN-60-SL-V1 ✅ (après consolidation)
- RUN-40-FARTLEK-V1 ✅

- FB-30-DENSITÉ-V1 ✅

**Principes** : Qualité structurée, volume montant, gestion de la charge.

- RUN-30-RECUP-V1 ✅ (prioritaire)
- RUN-40-EF-V1 ✅ (base stabilisée)
- RUN-60-SL-V1 ⚠️ (éviter en diète active)

- FB-40-FORCE-V1 ❌ (trop lourd en diète)

**Principes** : Priorité récupération, volume modéré, stabilisation pondérale, pas de double intensité.

## 4. Matrice de sélection : aide à la décision

### Scénario 1 : Semaine normale, accumulation de base
```
Décision : volume modéré, pas de qualité spéciale
→ RUN-40-EF-V1 + FB-30-BASE-V1 + RUN-40-EF-V1
ou RUN-40-EF-V1 + FB-30-MOBI-RENFO-V1 + RUN-30-RECUP-V1
```

### Scénario 2 : Semaine avec séance qualité running
```
Décision : 1 séance qualité running + soutien running
→ RUN-40-TEMPO-FRACTIONNE-V1 + RUN-40-EF-V1 + RUN-30-RECUP-V1
ou RUN-40-FARTLEK-V1 + RUN-40-EF-V1 + FB-30-BASE-V1
```

### Scénario 3 : Phase diète active, réduction de charge
```
Décision : priorité récupération, volume limité, pas d'effort intense
→ RUN-40-EF-V1 + FB-30-MOBI-RENFO-V1 + RUN-30-RECUP-V1
ou RUN-40-EF-V1 + FB-30-BASE-V1 + RUN-30-RECUP-V1
```

### Scénario 4 : Fatigue, surmenage signal
```
Décision : réduction immédiate
→ RUN-30-RECUP-V1 + FB-30-MOBI-RENFO-V1 + repos
ou RUN-30-RECUP-V1 seul
```

### Scénario 5 : Consolidation force, volume élevé
```
Décision : renforcement + running, pas de diète
→ RUN-40-EF-V1 + FB-40-FORCE-V1 + RUN-40-EF-V1
ou RUN-40-EF-V1 + FB-30-DENSITÉ-V1 + RUN-40-FARTLEK-V1
```

## 5. Règles de co-occurrence (à respecter)

| Règle | Raison |
|-------|--------|
| **Jamais deux séances RPE 6–7 le même jour** | Surmenage, récupération insuffisante |
| **Jamais RUN-45-SEUIL en diète active** | Stress métabolique cumulé, risque de fatigue |
| **Pas de RUN-60-SL en diète active** | Durée + diète = stress glycémique excessif |
| **RUN-40-TEMPO-FRACTIONNE : 1×/semaine max** | Intensité concentrée, pas d'accumulation |
| **RUN-45-SEUIL : 1×/semaine max** | Intensité maximale, créneau strict |
| **FB-40-FORCE après RUN-30-RECUP min** | Éviter superposition fatigue neuromusculaire |
| **Pas de FB-40-FORCE jour avant course longue** | Fatigue résiduelle, altération mécanique |

## 7. Notes méthodologiques

### Utilisation de ce registre par PS-PLAN-08
1. Recevoir la décision stratégique (ex: "semaine diète, récupération, 3 séances").
2. Interroger ce registre avec les critères (phase, charge, objectif).
3. Proposer une sélection d'activités justifiée.
4. **Signaler si le registre est insuffisant** (ex: "Impossible de sélectionner, car le profil XYZ n'existe pas").
5. Documenter la sélection dans le plan généré.

### Maintenance
- **Ajout** : créer la fiche activité + ajouter ligne à ce registre.
- **Modification** : mettre à jour le registre (impact PS-SCR-06 dans R_HEBDO).
- **Dépréciation** : conserver historique, marquer "V1 dépréciée" dans le registre.
